# Command-Sentinel - 危险命令检测工具
一款轻量级前端工具，用于检测 Windows/Linux 系统下的危险命令，支持 Base64 解码分析、系统关键路径检测、未知命令识别，同时适配 URL 参数调用（API 模式）。

## ⚠️ 重要提醒（必看）
1. 本工具仅为**文本层面的静态规则检测**，**并非云沙箱/动态分析工具**，无法检测二进制文件（.exe/.elf/bin 等）的实际行为；
2. 对于编译后的二进制代理、无文件攻击（Fileless）、内存执行类恶意行为，本工具无检测能力；
3. 工具仅能识别命令文本中的风险特征，无法对抗多层混淆/加密、Living off the Land（LotL）等高级攻击手段；
4. 本工具仅作为**辅助检测手段**，不可替代专业的终端防护（EDR/XDR）、沙箱分析、威胁情报等安全方案。

## 功能特性
### 核心检测能力
1. **多维度风险识别**
   - 🚨 极其危险：如 `rm -rf /`、`rmdir -q -s C:\`、磁盘格式化等致命命令
   - ⚠️ 危险：删除/修改系统关键路径（System32、/boot/、/etc/ 等）、强制删除文件/进程
   - ⚡ 风险：Base64 编码命令、远程网址/下载、未知二进制代理、编码转换/混淆命令
2. **智能解码分析**
   - 自动识别 Base64 编码命令并解码，递归检测解码后的内容风险
   - 支持 PowerShell 编码命令（`-EncodedCommand`）识别
3. **未知命令检测**
   - 基于白名单过滤系统合法工具（top/rm/rd/cd/ls/cat 等），识别陌生二进制代理程序名（仅检测名称，无法分析二进制内容）
4. **双系统适配**
   - 区分 Windows/Linux 命令体系，针对性检测各自的危险特征
5. **API 模式支持**
   - 解析 URL 中的 `command` 参数，自动检测并输出结构化结果

### 检测覆盖场景
| 检测类型                | Windows 特征示例                | Linux 特征示例                  |
|-------------------------|---------------------------------|---------------------------------|
| 致命删除命令            | `rmdir -q -s C:\`、`del /f System32` | `rm -rf /`、`rm -rf /boot/`     |
| 系统关键路径            | `C:\`、`System32`、`%WINDIR%`   | `/system/`、`/root/`、`/etc/`   |
| 编码/混淆命令           | `powershell -EncodedCommand`    | `echo YWJj | base64 -d`         |
| 远程操作                | `iwr http://mal.com`、`certutil` | `curl http://mal.com`、`wget`   |
| 未知二进制代理（仅名称） | `C:\tmp\hack.exe`               | `./unknown_agent`               |

## 快速开始
### 1. 部署运行
#### 方式1：本地静态服务器（推荐，无跨域问题）
```bash
# 1. 安装 Node.js 静态服务器（仅需一次）
npm install -g http-server

# 2. 进入工具目录（包含 index.html/style.css/script.js）
cd /path/to/Command-Sentinel

# 3. 启动服务器
http-server -p 8080

# 4. 访问工具
# 浏览器打开：http://127.0.0.1:8080
```

#### 方式2：直接本地运行（无需服务器）
- 将 `rules.json` 规则内嵌到 `script.js`（已提供内嵌版本）
- 直接双击 `index.html` 即可运行（无跨域限制）

### 2. 使用方法
#### 常规模式
1. 打开工具页面，选择「Windows 命令检测」或「Linux 命令检测」标签
2. 在文本框中输入待检测的命令**文本**（无法输入/检测二进制文件）
3. 点击「检测风险」按钮，查看检测结果（安全/风险/危险/极其危险）

#### API 模式（URL 参数调用）
直接在 URL 中传入 `command` 参数（支持 URL 编码），工具自动检测：
```
# Linux 命令检测（默认）
http://127.0.0.1:8080/?command=rm%20-rf%20/

# Windows 命令检测（指定 os 参数）
http://127.0.0.1:8080/?os=windows&command=rmdir%20-q%20-s%20C:\\

# 普通命令检测
http://127.0.0.1:8080/?command=echo%20hello
```
- `command`：待检测的命令文本（需 URL 编码，仅支持文本，不支持二进制）
- `os`：可选，指定系统类型（`windows`/`linux`），默认 `linux`
- 检测结果会同时展示在页面和浏览器控制台（结构化 JSON）

## 自定义扩展
### 1. 扩展风险规则
修改 `script.js` 中的 `riskRules` 对象（内嵌版本）或 `rules.json`（服务器版本）：
```javascript
// 新增 Linux 风险规则示例
riskRules.linux.risk.push({
  regex: "wget\\s+.*\\.sh|curl\\s+.*\\.sh",
  message: "风险：检测到下载并执行 Shell 脚本命令"
});
```

### 2. 扩展命令白名单
修改 `commandWhitelist` 数组，添加/删除系统合法工具（仅文本名称匹配）：
```javascript
// 新增 Windows 白名单工具
commandWhitelist.windows.push('python', 'java', 'node');

// 新增 Linux 白名单工具
commandWhitelist.linux.push('docker', 'kubectl', 'python3');
```

### 3. 调整风险等级
修改检测逻辑中的优先级，或调整 `detectRisk` 函数中的等级判定规则。

## 技术说明
### 文件结构
```
Command-Sentinel/
├── index.html       # 页面结构（Windows/Linux 标签、输入框、结果展示）
├── style.css        # 页面样式（响应式布局、风险等级配色）
├── script.js        # 核心逻辑（规则检测、Base64 解码、URL 参数解析）
└── rules.json       # 风险规则配置（服务器版本，可选）
```

### 核心函数说明
| 函数名                | 功能                                  | 局限性                          |
|-----------------------|---------------------------------------|---------------------------------|
| `detectRisk()`        | 核心风险检测函数，整合所有检测规则    | 仅检测文本特征，无动态执行分析  |
| `extractExecutables()`| 提取命令中的可执行程序名              | 仅提取名称，无法分析二进制内容  |
| `detectUnknownCommand()` | 对比白名单，检测未知命令名称        | 仅名称匹配，无法识别合法工具的恶意使用 |
| `base64Decode()`      | Base64 解码，处理 URL 安全编码/填充符 | 仅解码文本，无法处理二进制编码  |
| URL 参数检查模块      | 解析 URL 中的 command 参数，自动检测  | 仅支持文本参数，不支持二进制文件 |

## 局限性与补充说明
1. **核心局限**：
   - 仅能检测**文本命令**的静态特征，无法分析二进制文件（.bin/.exe/.elf 等）的实际行为；
   - 无法对抗无文件攻击（Fileless）、内存执行、多层加密/混淆的恶意命令；
   - 未知命令检测仅匹配程序名，无法识别“合法工具+恶意参数”的攻击（如 `certutil.exe` 下载恶意文件）。
2. **防御建议**：
   - 本工具仅为「文本命令的初步筛查」，需结合云沙箱（动态分析二进制）、EDR/XDR（终端行为监控）、威胁情报（恶意哈希/域名识别）形成完整防护；
   - 对于可疑二进制文件，需上传至专业沙箱平台（如 VirusTotal、Any.Run）进行动态分析。
3. **误判处理**：
   - 若出现误判，可扩展命令白名单或调整正则规则，优化检测精度；
   - 白名单仅覆盖常见合法工具，可根据业务场景补充行业专属工具名。
